Models:

Rule-Based Model

In this model, we break potential conflict identification into two steps. First, we retrieve information from the contract in order to identify the parties involved and the norms it contains. Second, we use such information to compare norms and identify potential conflicts.

In the information retrieval step, we use rules to identify (in the first paragraph of the contract) the parties in the contract. We identify norms within the text simply using a set of modal verbs, thus, if a modal verb exists in a sentence, we classify it as a norm. Finally, we compare the parties, the modal verb (and its deontic meaning), and the norm action and, using conflict rules, we identify which ones are potential conflicts.

Check our paper for more details: Identifying Potential Conflicts between Norms in Contracts

CNN Model

In this model, we use a CNN to process norm pairs and classify them as conflicts. We use a matrix representation that computes a character-level similarity between two norms as input to the CNN. To train our model, we use a set of conflicting norm pairs manually labeled and a set of non-conflicting norm pairs.

For further details, check our paper: Norm Conflict Identification using Deep Learning.

Embedding Model

In this model, we use a embedding representation of sentences to compare norms and identify conflicts. We use Sent2Vec to turn each norm into a latent vector, then giving a data set of conflicting norms, we generate an offset that represent the concept of conflict. We generate such offset by calculating the mean vector from the sum of all subtractions between conflicting norm vectors. To identify a new conflict, we use get the resulting vector from two norms and compare such vector with our offset. Given a threshold, we say that if the distance between the vector and the offset is below the threshold it is a conflict, otherwise it is not.

For further details, check our paper: Norm Conflict Identification using Vector Space Offsets
